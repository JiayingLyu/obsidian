### Page 01: MindVLA-U1：VLA与VA的能力差距在于接口设计

这次分享的论文是 MindVLA-U1。这篇工作探讨的核心问题是：在端到端自动驾驶领域，现有的 VLA（视觉-语言-动作）模型与纯粹的 VA（视觉-动作）模型之间存在闭环控制性能的差距。论文提出，这种差距的主要原因不在于模型参数规模，而在于多模态架构的接口设计存在错位。

首先看现有的 VLA 架构存在的问题。为了引入语言能力，先前的模型通常将动作信息离散化为 Token，然后再通过扩散去噪或其他方式进行控制。这种处理路径相对冗长。在离散化和去噪的过程中，原始的高精度几何信息与时序细节容易被压缩或破坏，导致物理控制精度下降。此外，模型在训练阶段的机制与实际部署时的闭环执行环境存在不一致。这也是为什么在同等算力条件下，直接输出连续动作的纯 VA 模型在闭环控制上的表现通常更优。

针对这种接口错位，MindVLA-U1 提出了一种端到端的流式接口设计。在底层架构上，它将视觉特征、自我状态、语言指令以及历史动作统一编码为连续的流式隐变量（Continuous Latent）。模型摒弃了离散 Token 的转换过程，利用单一的流式序列直接驱动后端的扩散策略头。结合流式因果建模与滑动窗口机制，这种设计天然对齐了实际驾驶中的闭环和长时依赖需求，减少了中间环节的信息损耗。

在解决动作精度问题的同时，该架构保留了语言模型的显式语义和可解释性。其动作头直接输出连续的物理轨迹或控制序列，能够与现有的主流底层执行器兼容。这使得模型在接收指令、调整驾驶风格等方面的可控性得到提升。右侧的雷达图对比展示了这一点：MindVLA-U1 在连续动作、流式连续性以及语言控制等维度上均实现了覆盖。

在具体的测试指标上，该架构在 WOD-E2E 基准测试的验证集上取得了 8.20 的 RFS 得分，测试集得分为 7.87。在推理效率方面，1B 参数规模下只需两步扩散即可达到约 16 FPS 的运行速度，说明该架构在保证规划精度的同时，能够满足实际部署的低延迟要求。

### Page 02: 为什么 VA 强但不够：连续控制强，显式语义弱

这一页具体对比了 VA、先前的 VLA 以及 MindVLA-U1 在各项核心维度上的能力差异，并解释了为什么自动驾驶既需要连续控制，也需要显式的语言语义。

首先分析 VA 模型。VA 模型的优势在于底层控制。它采用端到端的连续映射，能够实现厘米级的动作精度。同时，由于计算路径直接，其推理延迟通常可以控制在 20 到 50 毫秒之间，且流式表现稳定，非常适合处理实时驾驶任务。但是，VA 模型的内部特征是隐式的。它缺乏语言接口，决策过程缺乏可解释性；在面对未见过的长尾场景时，难以利用显式的语义常识进行逻辑推理或组合泛化。

为了弥补语义理解的不足，先前的 VLA 模型引入了语言大模型。但在实际应用中，它们多采用固定长度的分段（Chunk）或离散序列来处理视频和动作。这导致了两个主要问题：一是连续控制精度受到离散化的限制，误差容易随时间累积并产生时序漂移；二是处理多帧序列时存在较高的计算冗余，缺乏自适应计算能力。此外，这些模型中的语言到动作的控制往往只停留在浅层对齐，处理复杂指令的能力有限。

MindVLA-U1 的设计逻辑是整合两者的优势。在动作端，它保留了 VA 的连续控制特性和统一的流式建模，确保了动作输出的精度、低延迟以及状态的连贯性，减少了长时序下的轨迹漂移。在语义端，它提供了一个自然的语言接口，不仅使模型的行为具备可解释性，还能将复杂的自然语言指令直接转化为动作控制条件。

此外，在计算资源分配上，统一架构支持更灵活的上下文管理，能够根据任务需求进行自适应计算和早期退出（early exit），提高了整体的资源利用效率。总体而言，该架构试图证明，通过正确的接口设计，模型可以在不牺牲底层执行精度的前提下，有效整合高层语义理解与泛化能力。

### Page 03: 现有 VLA 的三类接口错配，解释了 precision floor 和 chunk discontinuity

本页主要说明现有 VLA 架构中存在的三类接口错配。论文指出，VLA 与 VA 模型之间的性能差距主要来源于接口层的设计，而非语言模型本身的能力限制。

第一类是动作接口错配。现有的 VLA 模型通常将连续动作离散化为 Token，或者通过分离的动作专家网络（separate action expert）进行映射。这种离散化和映射过程会引入量化误差，导致模型在输出端存在精度下限（Precision floor）。这使得模型在进行微小控制时精度受损，且单纯扩大模型规模也难以突破这一物理精度限制。

第二类是时序接口错配。传统模型倾向于使用固定长度的块（fixed chunks）来处理视频序列。这种方式会读取相邻帧中高度冗余的 Token，增加计算开销。同时，在不同 Chunk 的交界处，模型的上下文状态容易被重置或割裂，导致车辆在时序边界上出现行为不连续（boundary discontinuity）的问题。

第三类是语言-动作接口错配。尽管现有模型包含语言解码头，但缺乏一条从语言指令到连续动作空间的可度量控制路径。这导致模型的语言理解能力与底层的动作执行环节存在物理脱节，难以实现可验证的语言条件控制。

针对上述三个问题，MindVLA-U1 确立了相应的接口修正方向：采用连续动作输出以消除量化误差；使用流式记忆（Streaming memory）替代固定 Chunk 以维持状态的连续性；引入 Intent-CFG 建立语言到动作的控制路径；并设计快慢上下文管理以平衡计算效率。

### Page 04: 统一共享 backbone：语言保持 AR，动作保持连续，两者共享同一表示

本页展示了 MindVLA-U1 统一共享骨干网络（Unified shared backbone）的具体架构设计。该设计的核心逻辑是使动作 Token 与其他上下文 Token 进入完全相同的自注意力计算空间。

整体执行机制可以拆解为四个关键环节。首先是构建统一的 Token 集合。多模态输入（包括视觉、自我状态、语言指令、历史记忆）与连续的动作噪声 Token，会按照时间顺序拼接为同一个输入序列。

其次是共享 Transformer 骨干。上述序列会输入到同一组 Transformer 层中。在这个阶段，所有的 Token 进行混合交互，完全共享自注意力（Self-Attention）机制与前馈神经网络（FFN）的权重。

第三是两类读出头并行解码。在网络的最顶层，模型从同一个隐藏层表示 $H$ 中引出两个独立的解码头。其中，LM Head 采用自回归（AR）方式预测下一个离散的语言 Token；而 Flow-Matching MLP Head 则负责预测连续的速度场（Velocity field），用于生成动作轨迹。两者互不干扰。

这种设计的技术优势在于，它使得语言目标与动作生成在底层共享了梯度与特征表示，增强了跨模态的对齐效果，并且在结构上省去了独立的动作专家模块。相比于先前将动作离散化或采用分段建模的 VLA 做法，MindVLA-U1 在保持文本语义连贯性的同时，通过 Flow-matching 机制确保了动作输出的平滑与精细。

此外，图中中间部分的注意力掩码（Mask）配置展示了该架构的灵活性。由于所有 Token 都在同一个自注意力池中，模型仅需通过调整掩码的可见性，即可在推理时切换不同的执行顺序，如先执行语言推理（VQA-First）或先执行动作降噪（Action-First），而无需改变网络的主体结构。

### Page 05: Flow-matching action head：用连续扩散式轨迹绕开离散 token 精度底板

本页介绍动作输出端的具体架构。为了避免离散 Token 带来的量化误差，MindVLA-U1 采用 Flow-matching 机制在连续空间中生成物理轨迹。

首先看核心公式。该机制在纯高斯噪声 $\epsilon$ 和干净的真实轨迹 $x_0$ 之间构建线性插值，表示为 $x_t = t\epsilon + (1-t)x_0$。在这种连续时间建模下，模型的前向任务是预测一个速度场（velocity field） $v_\theta(x_t, t)$，使其逼近真实的导数 $\frac{dx_t}{dt}$。

在推理阶段，系统从高斯噪声出发，利用 ODE（常微分方程）求解器进行数值积分。具体操作是沿时间步从 $t=1$ 积分至 $t=0$，将速度场还原为连续的物理轨迹。右侧的 Figure 6 以俯视图形式展示了这一数值积分过程：随着 Euler 积分步的推进，蓝色的预测轨迹逐步收敛并贴合绿色的真实轨迹。

在具体参数配置上，模型在 4Hz 频率下预测未来 5 秒的 20 个连续航点。右图为了可视化细节展示了 5 步积分过程，但在实际部署中，模型仅需 2 个 Euler 步即可达到所需的控制精度，满足了自动驾驶的低延迟需求。

### Page 06: Streaming Memory：把时序上下文从“多帧堆 token”改成“可学习 FIFO 状态”

本页说明处理时序上下文的 Streaming Memory（流式记忆）机制。针对传统固定长度 Chunk 处理方式造成的时序割裂与计算冗余，该机制引入了逐帧流式输入与有限状态更新设计。

整个机制在每个时间步 $t$ 分为三个执行环节。
第一步是记忆读取与对齐。系统提取上一时刻的紧凑记忆 Token $M_{t-1}$。由于车辆处于运动状态，这些记忆需经过 $SE(2)$ 刚体运动对齐，转换至当前的自车坐标系下，生成对齐后的记忆 $\hat{M}_{t-1}$。
第二步是前向计算。当前的单帧多视角图像 $V_t$ 与对齐后的记忆 $\hat{M}_{t-1}$ 共同输入 Transformer 模型，输出当前帧的预测结果。
第三步是记忆更新。系统使用 Q-Former 结构，通过可学习的 Query 从模型的输出中提取最新特征，生成新的记忆 Token $M_t^{new}$。新记忆被压入 FIFO（先进先出）队列，同时丢弃最老的一组记忆，以此保证每帧的计算量具有明确上限（bounded compute）。

该机制的一个关键设计是，梯度可以跨帧流经 Q-Former，当前帧的损失会监督上一帧记忆的写入过程。左下角的消融实验结果表明，采用流式记忆架构（Streaming + memory）后，模型不仅在 RFS 上取得了 7.83 的最优成绩，同时在 25 秒的长序列评估中，将 ADE 误差降至 1.50m，验证了其在长时一致性上的有效性。

### Page 07: Intent-CFG：第一次把 language-side intent 变成可测的连续动作控制信号

本页介绍 MindVLA-U1 如何建立从语言理解到连续动作生成的控制路径。在以往的架构中，语言模型与动作输出之间缺乏可度量的因果联系 。Intent-CFG 机制解决了这一问题。

在具体机制上，VLM 的语言头首先通过标准的下一个词预测（Next-token prediction）输出当前场景的离散意图 Token $z$ 。该 Token 经过特征映射后，被残差添加到动作 MLP 预测速度场之前的时间嵌入（Time embedding）中 。

在推理阶段，模型利用无分类器引导（CFG）公式进行结构化混合 。模型会进行两次前向传播：一次计算受意图 $z$ 条件化的速度场 $v(z)$，另一次计算无条件速度场 $v(\emptyset)$ 。随后，通过引导强度系数 $s$，将两者线性混合为最终的速度场 $v_{cfg} = v(\emptyset) + s \cdot (v(z) - v(\emptyset))$ 。在扩散的反向积分过程中，这一混合速度场直接引导动作序列的生成 。

左下角的 Table 3 展示了该机制的定量影响 。在无意图（No intent）的基线设置下，模型的 RFS 为 7.83 。当引入模型自身预测的意图（NTP-predicted Intent-CFG）进行引导后，RFS 提升至 7.92 。这说明将语言端的意图作为连续动作的控制信号，能够有效提升整体的规划质量 。

### Page 08: Fast/Slow + MoT：同一 checkpoint 同时支持语义慢思考和动作快反射

本页说明该架构如何处理自动驾驶中复杂语义推理与低延迟实时控制之间的算力冲突。MindVLA-U1 通过调整注意力掩码（Attention mask）和引入混合专家（MoT）架构，在单一模型权重下实现了快慢系统的解耦 。

首先看注意力掩码的设计。由于所有 Token 共享同一个自注意力池，系统可以通过改变 Token 间的可见性来控制执行顺序 。例如，`vqa_first` 模式要求先进行语言解码，再将生成的答案作为动作去噪的条件；而 `action_only` 模式则完全屏蔽语言生成，提供一条严格缩短的前向传播路径，以实现快速反射 。

为了进一步提升快路径的物理效率，模型采用了 Mixture-of-Transformers (MoT) 结构 。在 FFN 层，网络被拆分为两个并行的专家组：上下文专家组处理视觉和语言 Token，动作专家组处理记忆、状态和动作 Token 。在自注意力阶段，这两组特征相互可见，保证了跨模态特征的融合 。

在 `action_only` 快路径执行时，语言相关的 Token 被物理移除，剩余的感知和动作 Token 直接路由到参数量较小的动作专家组中进行解码 。这避免了庞大的语言解码开销。右下角的表格数据显示，这种 MoT 架构不仅没有降低规划质量，反而有所提升。例如，采用 $\{V,L\}$ 与 $\{M,S,A\}$ 分组的 MoT 变体取得了 7.92 的 RFS 。这验证了该架构能在同一套参数下，有效平衡计算资源与控制需求 。

**论文一直采用的 MoT 架构吗？**

### 1. 论文中提到的“流式”（Streaming）的具体含义

在本文的架构中，“流式”（Streaming Paradigm）是指一种针对连续自动驾驶视频流的处理机制，用于替代传统的固定视频块（Fixed video-action chunks）处理方式。其核心特征包含以下几点：

* **逐帧处理（Framewise processing）**：在任意给定的时间步（Frame $i$），视觉-语言模型（VLM）仅接收当前的单帧多视角图像作为视觉输入，而不接收堆叠的多帧图像序列。


* **FIFO 记忆通道（Memory Channel）**：历史的时间上下文信息（Temporal context）不依赖于让模型重新对多帧图像计算注意力，而是通过一个有界限的先进先出（FIFO）记忆通道进行跨帧传递。当前帧会读取该通道中压缩的记忆特征 $m_i$。


* **计算成本有界与动作平滑**：由于避免了处理冗余的多帧输入数据，模型处理每一帧的计算开销是固定的（Bounded per-frame cost）。同时，由于记忆状态是帧间连续传递的，模型规划出的轨迹能够在整个驾驶过程中平滑演进，消除了传统分块处理在块边界处产生的动作不连续现象。


* **流式训练（Streaming training）**：在训练时，数据同样按照连续的帧顺序进行前向计算，梯度能够穿过传播模块在帧与帧之间流动。这意味着第 $i$ 帧产生的损失会直接监督模型在第 $i-1$ 帧写入记忆通道的内容。



### 2. 实验表格中各项 Metric 的含义说明

论文表格（如 Table 1 和 Table 2）中涉及的评估指标和标记具体含义如下：

* **AD-PT**：全称为 External Autonomous-Driving Pretraining，指该模型是否使用了外部的自动驾驶数据进行预训练。表格中的符号 $\checkmark$ 表示使用了外部预训练，$\times$ 表示未使用，$?$ 表示该方法原文献中未报告此信息。


* **L B/R**：代表 Language BLEU-4 和 ROUGE-L。这是两个自然语言处理中常用的评估指标，用于衡量模型的视觉问答（VQA）输出头在回答场景问题时的语言生成质量。


* **RFS-GT ADE 3/5**：指预测轨迹与日志记录的真实 Ground-Truth (GT) 轨迹之间的平均位移误差（Average Displacement Error）。数字 3/5 分别代表评估的是未来 3 秒和 5 秒时间窗口内的轨迹误差。


* **RFS-matched ADE 3/5**：指预测轨迹与 RFS 评估系统中提供的人类参考轨迹之间的平均位移误差。在 RFS 评估机制下，每个场景有三条人类标注的参考轨迹，该指标测量的是预测轨迹与这三条中最近的（最匹配的）一条参考轨迹在 3 秒和 5 秒时的误差距离。

以下是为您解答 RFS 的含义，并为您补充的强化学习（RL）部分汇报稿及插入建议。

### 1. RFS 的具体含义

**RFS (Rater Feedback Score，人类评分员反馈得分)** 是自动驾驶评估中的一个核心标量指标，也是 MindVLA-U1 强化学习阶段唯一的奖励信号。
它的评分机制是：针对模型预测出的一条轨迹，系统会将其与多位人类专家标注的参考轨迹进行对比。如果预测轨迹落在基于车速构建的“人类信任区域（Trust Region）”内，就能直接获得高分（满分 10 分）；如果偏离了该区域，得分会随偏离距离呈现指数级衰减，保底得分为 4 分。它反映的是轨迹是否符合人类驾驶的综合偏好与舒适度。

> 在探讨上述 RFS 得分从 7.83 提升至 8.20 的过程中，需要补充说明模型所采用的强化学习（RL）后训练机制。
> 在传统的自动驾驶强化学习中，通常需要设计复杂的奖励函数，包含各项平滑度、距离或碰撞惩罚等辅助项。而 MindVLA-U1 的 RL 策略极其简化：它剥离了所有的辅助奖励，**仅使用 RFS（人类评分员反馈得分）这一个标量作为奖励信号**。
> 在具体算法上，模型采用了 GRPO（组相对策略优化）算法。对于同一个训练场景，模型会并行生成 8 条不同的候选轨迹，分别计算它们的 RFS 得分，然后在这一组内进行相对比较和梯度更新。表现高于组内均值的轨迹被鼓励，低于均值的轨迹被抑制。
> 这一纯粹的 RL 过程揭示了一个重要的评估现象：经过强化学习优化后，模型预测轨迹与人类评分团认为的最优轨迹之间的误差（Rater-matched ADE）降低了；但它与数据集中记录的那唯一一条真实行驶轨迹（Ground-Truth）的距离反而略微变大了。这在算法逻辑上说明，仅优化 RFS 奖励，使得模型的驾驶策略从“单纯模仿日志中的某一种合法行为”，转向了“迎合多位人类专家的共识偏好”，这是模型在评估中能够超越单个人类参考线的底层原因。
