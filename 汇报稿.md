### Page 01: 首页

各位，今天我们来深度拆解 MindVLA-U1 这篇工作。目前端到端自动驾驶领域的一个核心痛点是：尽管大家都在向 VLA（视觉-语言-动作）架构演进，试图让模型同时具备看、想、说、做的能力，但在实际的闭环控制表现上，VLA 往往会被纯粹的 VA（仅视觉-动作）模型拉开差距。很多人将其归咎于语言模型的参数规模不够，但这篇论文一针见血地指出：VLA 跑不过 VA，根本原因在于**接口设计（Interface Design）的错位**。

当我们在系统中强行引入语言模块时，往往伴随着信息的强行离散化以及去噪过程中的精度破坏。MindVLA-U1 提出了一套端到端的流式统一接口，将视觉、状态、语言和历史动作统一编码为连续的隐空间特征（Latent），使用单一流式序列直接驱动扩散策略头。它没有任何中间的离散化重投影损耗，天然对齐了实时闭环控制的需求。

大家可以看右下角的 KPI。在统一接口的加持下，MindVLA-U1 在极具挑战的 WOD-E2E 基准上，不仅达到了 8.20 的 RFS 得分，更重要的是，在 1B 参数规模下仅需 2 步扩散，就能实现大约 16 FPS 的推理速度。这为后续通过数据和算力进行 Scale-up 提供了一个极其干净、高效的基础设施。

### Page 02: 背景对比 - 为什么 VA 强但不够？

在展开架构前，我们需要客观评估 VA 与现有 VLA 的优劣势。自动驾驶长尾问题的本质，要求系统既要有 VA 级别的连续动作精度，又要有 VLA 级别的显式语义理解。

纯 VA 模型强在什么地方？它的映射路径极其直接，从传感器直接回归到物理空间，厘米级控制精度极高，且计算路径短，延迟通常能压在 20-50ms。但它的致命弱点是“语义隐式”，模型内部是一个巨大的黑盒，它无法遵循复杂的自然语言指令，遇到没见过的长尾场景很难泛化。

现有的 VLA 虽然引入了语言头，但付出了巨大的代价。动作生成往往受限于离散化解码，存在微小控制误差累积；时序上多采用固定长度的视频片段（Chunk），容易产生漂移。MindVLA-U1 的破局点在于，它在底层设计上彻底融合了两者：用纯连续的控制保障精度，用统一流式建模消除边界割裂，同时完整保留了语言模型的显式语义表征和指令跟随能力。

### Page 03: 现有 VLA 的三类接口错配

之所以会产生前面的性能差距，作者归纳为现有 VLA 的三类“接口错配”。这三点是我们组内做架构设计时必须警惕的坑。

第一是**动作接口错配**。当前主流做法是把连续动作强行切分为离散的 Token，或者外挂一个独立的动作专家网络。只要引入量化映射，物理空间必然存在一个无法突破的精度底板（Precision Floor）。不管大模型参数扩到多大，微小控制与接触力控制依然受损。

第二是**时序接口错配**。大家习惯用固定长度的 Chunk 去吃多帧视频序列。这就带来两个灾难：一是相邻帧之间高度冗余，Token 开销极大；二是当执行完前一个 Chunk，切到下一个 Chunk 时，物理状态被强行割裂重置，导致车辆行为在边界处发生顿挫。

第三是**语言-动作接口错配**。现有网络虽然挂着个语言头，但语言和底层的物理动作之间根本没有一条可度量的因果路径。语言指令往往沦为无效的 Prompt，模型的“理解”与“执行”是脱节的。这三个错配，也正是 MindVLA-U1 后续提出流匹配、流式记忆和 Intent-CFG 的根本动机。

### Page 04: 统一共享 Backbone

这是整个模型的心脏：统一共享骨干网络。对比主流 VLA 要么让动作纯离散化、要么搞一条独立的动作旁路分支，MindVLA-U1 做到了一种近乎暴力的极简。

在输入端，视觉、自回归语言、历史记忆、自我状态，以及一堆代表连续轨迹的纯噪声动作 Token（Noisy Action Tokens），全部按时间顺序被拼接到同一个序列里。在模型内部，这些 Token 一起流经所有的 Transformer 层，完全共享自注意力（Self-Attention）和前馈网络（FFN）。

它的精妙之处在于**双头读出机制**。在网络的最顶端，语言 Token 对应位置挂着自回归（AR）语言头，负责预测下一个离散 Token；而动作 Token 对应位置挂着一个极轻量的 Flow-Matching MLP，用来预测连续的速度场。因为动作 Token 在前向传播时完全处于相同的自注意力池中，它天然吸饱了视觉和语言的上下文语义。语言目标与生成目标在底层共享表征与梯度，互不干扰，实现了模态的深度绑定。

### Page 05: Flow-matching 动作头

接下来看动作是如何具体生成的。既然我们认同离散 Token 会锁死控制精度，我们就必须回归连续空间建模。这里的动作头抛弃了传统的自回归，采用了与我们常在 DiT 等高级视频生成架构中见到的 Flow-matching（流匹配）机制。

简单看左侧的核心思想：系统将真实的干净轨迹 $x_0$ 和纯高斯噪声 $\epsilon$ 进行线性插值，构造出任意时刻的中间状态 $x_t$。网络的任务不是去预测离散坐标点，而是去拟合一个连续的速度场（Velocity Field）$v_\theta(x_t, t)$，让它逼近真实的导数 $\frac{dx_t}{dt}$。

到了推理阶段，我们从纯噪声开始，将其送入常微分方程（ODE）求解器，使用简单的欧拉法（Euler）进行数值积分。由于 Flow-matching 学习到的轨迹非常平滑且近乎直线，这里不需要像传统 Diffusion 那样跑几十步。在 4Hz 频率下生成未来 5 秒的 20 个连续航点，**仅需要 2 个 Euler 步**。图 6 虽然画了 5 步为了展示细节，但在实际部署和跑分中，2 步就已经足够，这对于自动驾驶的实时性至关重要。

### Page 06: Streaming Memory 流式记忆

为了彻底干掉前面提到的 Chunk 边界割裂问题，模型引入了 Streaming Memory。我们将过去“堆叠多帧图像 Token”的粗暴做法，升级成了“可学习的 FIFO 紧凑状态”。

在每一个时刻 $t$，系统只接收当前的单帧多视角图像 $V_t$，同时从 FIFO 队列中调出上一时刻浓缩的记忆 $M_{t-1}$。**这里有一个极其关键的物理转换**：因为车一直在开，上一帧的记忆坐标系已经过时了，必须经过 $SE(2)$ 刚体运动对齐，转换到当前自我视角的坐标系下，得到 $\hat{M}_{t-1}$。随后，图像与对齐后的记忆一起进入 Transformer 进行前向计算。

计算结束后，为了更新记忆，系统不保存原始图像，而是利用一个 **Q-Former** 结构，用少量的 Learnable Queries 从模型隐层中“榨取”出最有用的上下文特征，生成新的记忆写入 FIFO 队列尾部，并踢掉最老的记忆。大家看消融实验表格，在替换为流式记忆后，不仅 RFS 从 7.69 涨到了 7.83，在长达 25 秒的超长序列上，ADE 也能稳定下降到 1.50m，这种 Bounded Compute 带来了极强的长时序一致性收益。

### Page 07: Intent-CFG 意图引导

这一页是本文在“可控性”上最大的创新：Intent-CFG。怎么才能证明 VLM 生成的文本指令不是摆设，而是真的控制了方向盘？作者借用了我们在 AIGC 图像生成中非常熟悉的 Classifier-Free Guidance（CFG）无分类器引导机制。

前向计算时，VLM 首先自回归预测出一个离散的意图 Token $z$（比如：左转变道）。这个 Token 被提取成特征，注入到条件化速度场的预测中。同时，模型也会预测一个空集 $\emptyset$ 下的无条件速度场。

在 Flow-matching 的反向积分阶段，利用公式 $v_{cfg} = v(\emptyset) + s \cdot (v(z) - v(\emptyset))$ 进行线性外推混合。我们通过控制引导强度 $s$，强行把降噪轨迹朝着语言意图的方向推。在表 3 的实验中，仅仅引入这个模型自己预测的 Intent-CFG，就在没有任何外部数据预训练的情况下，将 RFS 强行拉升了 0.09。这彻底打通了从高级语义意图到连续物理控制的因果链路。

### Page 08: Fast/Slow 与 MoT 架构

我们常常面临一个悖论：自动驾驶既需要大模型做深度的安全长尾推理，又需要毫秒级的快速反射神经。MindVLA-U1 证明，这并不需要拆成两个模型，通过**底层架构级别的路由管理**就能完美解决。

这就是基于 Mixture-of-Transformers (MoT) 的快慢系统调度。我们在自注意力层使用掩码（Mask）控制先后顺序：vqa_first 模式让动作生成必须等待文本推理完成；而 action_only 模式则直接屏蔽文本查询。更深一层，MoT 机制将 FFN 拆分为“上下文专家（处理视觉/语言）”和窄带的“动作专家”。

这就意味着，当系统处于 Fast 路径（Action-only）时，极其消耗算力的上下文专家和语言解码通道在物理上被闲置或绕过，视觉与状态特征直接路由到轻量的动作专家组进行极速降噪。一套模型权重，实现了认知与反射的无缝切换。右下角的表格也证明，MoT 稀疏架构不仅没掉点，反而能在特定编排下（如 8.01 RFS）超越传统的 Dense 密集网络。

### Page 09: 整体实验结果

这些精巧的架构设计最终必须兑现到绝对的规划指标上。在 WOD-E2E 数据集上，验证了 MindVLA-U1 是真正能够闭合 VLA 与 VA 性能 Gap 的架构。

大家关注几个 KPI 数据：首先是在 Validation Split 上，经过简单的 RL 强化学习后训练，MindVLA-U1 达到了惊人的 **8.20 RFS**，这在开环指标上第一次超越了人类驾驶员的真实行车记录得分（8.13）。其次，在 Official Test 测试集上，它依然保持了最高的 7.87 RFS，同时达到了目前 SOTA 的 1.09m/2.66m (3s/5s) ADE 精度。值得强调的是，在 U1 对应的模型行中，没有任何外部自动驾驶预训练数据的加持，纯靠这套架构吃透了现有数据分布。

### Page 10: 延迟拆解与吞吐量

既然宣称能做实时闭环，就必须直面延迟问题。这里对延迟瀑布做了极度残酷的解剖。

以 batch size=1 在单张 H200 上测试，慢路径（即包含完整的 VQA 推理问答）耗时高达 2,594 ms，其中 **94% 的延迟被卡在自回归（AR）的文本解码上**（约 2,450 ms）。相比之下，完整的 2 步 Euler 动作扩散仅需不到 74 ms。

这暴露了一个巨大的工程红利：VLM 架构本身并不慢，慢的只是吐字过程。当我们切到 action_only 快路径，从注意力层面剥离文本解码后，模型在 1B 参数规模下直接跑到了 **15.55 到 16 FPS**。这个吞吐量已经完美对齐了当前顶尖的纯 VA 模型（如 RAP-DINO 的 17.68 FPS）。它用数据证明了，在 VLA 中保留高级语义中枢，绝不会成为压垮实时物理控制系统的性能包袱。

### Page 11: Scaling 与语言质量解耦

接下来讲一个具有行业前瞻性的反直觉结论。我们一直有个思维惯性：VLM 参数越大、语言能力越强，自动驾驶的性能就越好。但这篇工作的 Scaling 实验打破了这个迷思。

看这张折线图，从 0.8B 扩增到 9B，RFS 分数呈现出非单调的抖动，并非一条完美的 Scaling Law 曲线。最硬核的证据在左下角的表格：作者对比了 2B 模型的指令微调版（Instruct）和未经指令对齐的基座版（Base）。结果发现，Base 版的语言流畅度（BLEU 和 ROUGE 指标）发生了**约两倍的断崖式崩塌**，甚至没法好好说话，但底层物理控制的 **RFS 得分竟然基本保持在 7.91 的高位**。

这说明了什么？说明模型对物理世界的深层表征认知，与它表面在自然语言问答上的流畅度是**结构性解耦**的。这也提醒我们在做组内基座选型时：单纯堆叠大模型参数、追求极致的聊天评测分，对于自动驾驶控制上限的提升并不是线性等价的，动作接口容量和专属训练 Schedule 才是更关键的破局点。

### Page 12: 总结与局限性

最后做个收拢。MindVLA-U1 的核心贡献在于，它提供了一套高度一致的架构范式来填补 VLA 与 VA 之间的能力沟壑。
它的四大支柱紧密咬合：
第一，通过统一的 Transformer 骨干，让自回归语言和连续流匹配动作在同一个特征空间中生长。
第二，用包含空间对齐的 FIFO 流式记忆，取代了粗暴的多帧视频堆叠。
第三，通过 Intent-CFG 建立了一条从语义意图到物理轨迹的数学因果链路。
第四，利用 MoT 快慢通道，在同一个权重模型下解决了语义审视与极限运动反射的冲突。

当然，这篇论文也留下了需要我们后续攻克的局限性：当前的成绩完全基于 WOD-E2E 的开环（Open-loop）评测，缺乏真实的闭环环境泛化验证；它也没能跑出一条绝对干净的 Scaling Law 曲线；同时其庞大的多模态数据管线 MindLabel 也未被完全利用。但这套架构思想，对我们在设计下一代多模态物理控制基座时，具有极高的参考价值。
